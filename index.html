Python内存管理与垃圾回收

内存管理

Python对象

Python中一切皆对象，对象的类型和值都是储存在内存中的。

核心

池化技术

垃圾回收

Python的垃圾回收机制主要基于引用计数和分代回收。

引用计数

每个对象都有一个引用计数，表示该对象被引用的次数。当对象的引用计数为0时，表示该对象不再被使用，可以被回收。
Python中可通过sys.getrefcount()查看对象的引用计数。
但是引用计数不能解决循环引用的问题，因此需要分代回收。

分代回收

Python将对象分为三代，分别是0代、1代和2代。
所有新建的对象都是0代对象。

python-pandas

数据类型

  • 一维数据：Series
  • 二维数据：DataFrame

数据的查看方法包括：

  • head()：查看前n行
  • tail()：查看后n行
  • info()：查看数据的基本信息，列索引、数据类型、非空计数和内存信息
  • describe()：查看数据的基本统计信息，包括计数、均值、标准差、最小值、四分位数、最大值
  • shape：查看数据的大小，行数和列数
  • dtypes：查看数据类型
  • columns：查看列名
  • index：查看行索引

Pandas数据IO需要额外的库支持：如openpyxl

  • read_csv()：读取CSV文件
  • read_excel()：读取Excel文件

数据库

python中可以使用sqlalchemy或者pymysql连接数据库。

  • 使用sqlalchemy连接数据库
  • 使用pymysql连接数据库

pandas中的read_sql()函数可以读取数据库中的数据。

to_sql()函数可以将数据写入数据库。

数据选取

loc和iloc是pandas中用于数据选取的两种方法。

  • loc：基于标签的选取，可以选取单行、单列、多行、多列，以及切片。

  • iloc：基于位置的选取，可以选取单行、单列、多行、多列，以及切片。

  • 布尔索引

数据集成

concat()函数可以用于数据集成，将多个数据集拼接在一起。

insert()函数可以用于数据集成，将一个数据集插入到另一个数据集的指定位置。

merge()函数可以用于数据集成，将两个数据集按照某个键进行合并。

数据清洗

重复数据过滤方法有：

  • drop_duplicates()：删除重复数据
  • duplicated()：返回一个布尔值，表示每个数据是否重复

空数据过滤方法有：

  • dropna()：删除空数据
  • fillna()：填充空数据
  • isnull()：返回一个布尔值，表示每个数据是否为空
  • notnull()：返回一个布尔值，表示每个数据是否不为空

删除行和列的方法有：

  • drop()：删除行或列，axis=0表示删除行，axis=1表示删除列。

filter()函数可以用于数据过滤，返回符合条件的行。
可以指定标签或使用正则表达式。

常见的数据清洗还有过滤异常值、数据转换、数据标准化等。

数据转换

轴索引修改方法有：

  • set_axis()：设置轴索引
  • rename()：重命名轴索引
  • rename_axis()：重命名轴索引

值替换方法有：

  • replace()：替换值
  • replace_na()：替换空值
  • replace_inf()：替换无穷大
  • replace_nan()：替换NaN

map()函数可以用于series,不能用于dataframe.

apply()函数可以用于dataframe,但是只能对行列进行操作,支持series，可操作多个series。

applymap()函数可以用于dataframe,对整个dataframe进行操作。

transform()函数可以用于dataframe,对整个dataframe进行操作,可以对多列进行不同操作。

Python闭包

命名空间

命名空间是变量名和对象的映射关系。Python的命名空间包括：

  • 内置命名空间：Python自带，任何模块都可以访问，包含内置函数和异常的名称。
  • 全局命名空间：模块中定义的函数、类、模块级的变量和常量等。
  • 局部命名空间：函数中定义的变量和参数。

命名空间中的变量名没有任何关系。

命名空间的生命周期如下：

  • 内置命名空间：Python解释器启动时创建，直到Python解释器退出。
  • 全局命名空间：Python文件执行时创建，直到Python文件执行结束。
  • 局部命名空间：函数执行时创建，直到函数执行结束。

模块的属性和方法，都是定义在全局命名空间中。属性和global name之间是映射关系，共享同一个命名空间。

作用域

作用域是变量名可以被访问的区域。
作用域的搜索顺序：

  • 局部命名空间
  • 全局命名空间
  • 内置命名空间

对于变量名被声明为global的变量，那么所有的引用和赋值都会直接作用于全局命名空间。
并且只有在函数和类里面，才要定义或者修改全局变量时使用global。
函数外部，local作用域和global作用域拥有一样的命名空间，此时不需要声明global。

闭包

闭包就是能够读取外部函数变量的内部函数。

闭包的作用有两个：

  • 可以访问外部函数的变量
  • 可以延长外部函数的变量生命周期（持久化）

闭包的优点总结：

  • 局部变量无法共享和长久保存，全局变量可能造成变量污染，闭包可以解决这个问题。
  • 闭包可以使得函数内部变量不被释放，可以延长变量的生命周期，不会在外层函数执行完就被释放。
  • 闭包可以减少全局变量的使用，可以隐藏内部实现细节，可以实现函数式编程。
  • 闭包可以实现装饰器，装饰器是用于扩展函数功能的一种方式。

使用闭包的注意点：

  • 闭包可以保存外部函数的变量这个特性主要基于函数在编译时固定了变量的地址，闭包无法改变这个地址，如果在闭包中更改了这个变量，闭包会新增一个局部变量。nonlocal声明可以改变这个地址。
  • 闭包的使用一定程度上加重了内存的占用

GBDT

1. GBDT的通俗理解

GBDT继承了Boosting的思路，通过多棵决策树的加权组合，累加得到最终结果。在运算过程中，每棵树利用上一棵树的残差作为新的训练数据，通过不断迭代获得最终结果。

1.1 决策树

GBDT中，每棵决策树都是一棵回归树，回归树的输出值是连续值，而分类树的输出值是离散值。
对于每棵回归树，首先初始化一个弱学习器，然后迭代T次，在每次迭代中，对于样本$i$计算其残差$r_{i,t}$，然后利用残差$r_{i,t},x_i$拟合一个CART回归树，并计算最佳拟合值用来更新弱学习器，最终得到一个强学习器。

1.2 梯度提升

在每轮的残差计算中，GBDT利用损失函数的负梯度作为残差的近似。（GBDT的损失函数是平方损失函数）
常用的回归树的损失函数有平方损失函数、绝对损失函数、Huber损失函数和分位数损失函数。Huber损失函数是平方损失函数和绝对损失函数的结合，在误差较小时表现为平方损失，在误差较大时表现为绝对损失。分位数损失函数在处理正负样本不均衡时非常有用,二者的作用是的减少异常值对损失函数的影响。
常用的分类树的损失函数有指数损失函数、对数损失函数和Hinge损失函数。

1.3 正则化

GBDT的正则化手段有：

  • 调整步长
  • 子采样
  • CART决策树剪枝
  • Early Stopping

2. scikit-learn中的GBDT

2.1 参数

boosting参数：

  • n_estimators：弱学习器的数量
  • learning_rate：步长
  • loss：损失函数
  • init：初始化弱学习器
  • subsample：子采样
  • alpha：Huber损失函数中的参数(quantile)

决策树参数：

  • max_depth：最大深度,默认值为3
  • min_samples_split：最小分割样本数,少于这个数目，则不分割
  • min_samples_leaf：最小叶子节点样本数,少于这个数目，则会剪枝
  • max_features：最大特征数,默认值为None，表示使用所有特征，另外比如max_features=sqrt，则表示随机选择sqrt(n_features)个特征
  • max_leaf_nodes：最大叶子节点数,默认值为None，表示不限制叶子节点数

Python基础知识总结

1. Python 拷贝

在python中，使用已知变量对新变量赋值有三种方式：

  1. 直接赋值：即新变量与已知变量指向同一个内存地址
  2. 浅拷贝
  3. 深拷贝

对于深拷贝和浅拷贝，需要区分已知变量是可变对象还是不可变对象。
对于不可变对象，深拷贝和浅拷贝没有区别，都是直接赋值，即新变量与已知变量指向同一个内存地址。
对于可变对象，深拷贝和浅拷贝有区别。
浅拷贝会创建一个新的对象，但是新对象中的元素仍然指向与已知对象相同的内存地址。
深拷贝会创建一个新的对象，并且新对象中的元素也会创建新的内存地址。

笔者理解：
从内存角度理解，直接赋值、浅拷贝到深拷贝的过程是把已知变量copy并创造新变量的程度区别。
直接赋值不进行拷贝，浅拷贝只copy外层，而指向的内层的内存地址没有改变，深拷贝则是同时创造了新的外层和内层的copy（考虑对象可变性）。

2. Python对象

常用对象：元组字典列表集合
他们的异同点有：

  1. 元组是不可变对象，字典、列表、集合是可变对象
  2. 字典、列表、集合是可迭代对象，元组是序列对象
  3. 字典、列表、集合是容器对象，元组是扁平对象

3. 迭代器

迭代器是实现了__iter__和__next__方法的对象。
列表、字典、集合、元组、字符串都是可迭代对象，但是它们不是迭代器。
可迭代对象的特点是实现了__iter__方法，这个方法返回一个迭代器，迭代器可以按照一定顺序生成下一个元素。
Python内置的iter()函数可以接受一个可迭代对象，并返回一个迭代器。迭代器使用next()函数来获取下一个元素，当迭代器没有元素时，会抛出StopIteration异常。Python中的for循环就是通过迭代器来实现的。

4. 生成器

生成器是一种特殊的迭代器，在迭代的过程中动态的生成值。
生成器使用yield关键字来定义，当生成器的代码执行到yield时，会返回一个值，并且保存当前的执行状态，下次调用next()函数时，会从上次保存的执行状态继续执行。
因此生成器最大的特点在于节省内存，因此生成器适合用于需要大量数据处理的场景。
